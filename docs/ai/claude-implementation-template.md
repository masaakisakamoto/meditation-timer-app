# Claude Code 実装依頼テンプレート

## 使い方

1. このテンプレートをコピーする
2. `【】` の部分を埋める
3. Claude Code に貼り付けて送信する

---

## テンプレート（コピペ用）

```
## 目的
【実装したい機能・変更内容を1〜3行で書く】

---

## 変更可能ファイル

以下のファイルのみ変更してください:

- 【ファイルパス1】
- 【ファイルパス2】

---

## 変更禁止ファイル

以下のファイルは絶対に変更しないでください:

- 【禁止ファイル1（例: src/screens/TimerStop.tsx）】
- 【禁止ファイル2（例: Timerコンポーネント）】
- 親コンポーネントへの props 追加も禁止です

---

## 最初にやること

いきなり実装しないでください。

まず対象ファイルの現在のコードを読み、
以下を整理してから回答してください:

① 現在受け取っている props / 入力の整理
② その情報だけで実現できる範囲
③ 実現できない情報（今回は無理に実装しない）

---

## 制約

- 既存 props 以外は追加しない
- diff は最小にする
- 既存ロジック（タイマー・音声・通知）に影響を与えない
- `npm run check` が通ること（prettier + tsc）
- none / undefined / null が来ても落ちないこと

---

## 仕様詳細

【仕様を箇条書きで書く】

---

## スタイル方針（UI変更の場合）

- fontFamily: ZenMaruGothicMedium
- 既存スタイルと統一すること
- 【その他スタイル指定があれば書く】

---

## 最終出力フォーマット

以下の順で回答してください:

① 現在の props / 入力の整理
② 既存情報だけで実現できる範囲と実現できない範囲
③ 実装方針の要約（変更内容を箇条書き）
④ 変更ファイル一覧
⑤ 最終コード（全文 or diff）
```

---

## 補足ルール

### コードを書く前に必ず確認させること

- 対象ファイルを `Read` で読んでから実装を始めること
- 読んだ内容を整理してから実装すること（先に整理を回答させる）

### 暴走防止

- 「変更ファイルは〜のみ」と明示する
- 「最初にやること」セクションで先読みを強制する
- 「diff は最小にする」を必ず入れる

### MitterTimer 固有の注意事項

- React Native / Expo / iOS中心
- タイマーロジック（TimerStop.tsx）は特に慎重に扱う
- 音声（expo-audio）・通知（expo-notifications）のロジックは原則触らない
- `npm run check` = `prettier --check` + `tsc --noEmit`
- EAS Build / TestFlight でリリースするため、挙動破壊は避ける
