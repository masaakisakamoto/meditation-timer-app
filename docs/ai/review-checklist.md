# Claude提案 レビューチェックリスト

## 目的

Claude Code が出力した実装・diff を、ChatGPT または人間がレビューするときの
**Yes / No 判定基準を固定化**するチェックリスト。

全項目を確認してから最終判定を出すこと。

---

## 基本確認項目

| # | 確認内容 | Yes / No |
|---|---|---|
| 1 | 変更ファイルが指定範囲内だけか | |
| 2 | 変更禁止ファイルに触れていないか | |
| 3 | 親コンポーネントを勝手に変更していないか | |
| 4 | props を勝手に追加・変更していないか | |
| 5 | 既存ロジック（タイマー・音声・通知）に影響していないか | |
| 6 | diff が要求より大きくなっていないか | |
| 7 | 仕様書に記載のない機能を追加していないか | |
| 8 | TypeScript strict に通るか（`npm run check` 相当） | |

## No 判定基準（即却下）

以下のいずれかに該当する場合は **No（即却下）**:

- 変更禁止ファイルが変更されている
- 親コンポーネントへの props 追加が含まれている
- タイマーロジック・音声・通知のロジックが変更されている
- `npm run check` が通らない変更が含まれている
- 仕様に記載のない機能が追加されている
- diff が明らかに要求の範囲を超えている

---

## 修正指示が必要な場合（条件付き Yes）

軽微な問題がある場合は修正指示を付けて返却:

- スタイルの統一がずれている → 修正箇所を具体的に指摘
- 不要なコメントや console.log が含まれている → 削除を指示
- 表示ロジックの分岐が不完全 → ケースを追加指示

```
- （簡潔に）
```
