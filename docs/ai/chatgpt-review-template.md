## レビュー観点

以下の観点でレビューしてください:

1. **制約違反**: 変更禁止ファイルが変更されていないか / props が勝手に増えていないか
2. **仕様逸脱**: 依頼した機能以外が混入していないか
3. **diffの妥当性**: 変更量が依頼内容に対して適切か / 過剰な変更がないか
4. **型・依存の安全性**: TypeScript strict に問題ないか / 循環依存がないか
5. **UI/UX の妥当性**: 既存UIを壊していないか / none・null の安全処理があるか
6. **ロジックの安全性**: タイマー・音声・通知のロジックに影響していないか

---

## 判定依頼

上記観点を確認した上で、以下のいずれかで判定してください:

**Yes**: そのままマージ可能
**条件付きYes**: 軽微な修正後にマージ可能
**No**: 再実装が必要

## 補足：よくある制約違反パターン

ChatGPT にレビューさせるとき、特に以下を見てもらうと効果的:

| パターン | 確認ポイント |
|---|---|
| 親コンポーネント変更 | TimerStop.tsx / App.tsx に変更が混入していないか |
| props 追加 | 変更可能ファイル以外のコンポーネントのProps型が変わっていないか |
| ロジック混入 | useEffect / useCallback / Ref の変更が含まれていないか |
| スタイル崩れ | width / flex / position の変更が他レイアウトに影響しないか |
| null安全 | `??` / `?.` なしで MeditationType や配列インデックスにアクセスしていないか |

## MitterTimer 固有の注意事項

- `TimerStop.tsx` のベル発火ロジック・AppState処理は特に変更リスクが高い
- `expo-audio` の `useAudioPlayer` は複数インスタンス管理に注意
- `expo-notifications` のスケジュール処理は副作用が大きい
- フォントは `ZenMaruGothicMedium` / `ZenMaruGothic-Medium` の表記揺れに注意
